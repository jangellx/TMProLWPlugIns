Demos

Building the Demos

Source code for several Spaceball demos (Widget, CityFly, sbtest32, and sbtstmfc) is provided to show examples of programming for the Spaceball device. The demo makefiles were created using the Visual C++ development environment, which allows for either Release or Debug builds of the program on either of the two architectures. To build a demo follow these steps:

  1. Load the makefile into the Microsoft Developers Studio (File > Open Workspace > List Files of Type > Makefiles). 
  2. Select the type of build you want.
  3. Compile.

The executable should be located in the platform-specific bin directory, as follows:

Alpha

%SDK%\demos\demoname\bin\win32a\debug (or release)

Intel

%SDK%\demos\demoname\bin\win32i\debug (or release)

CityFly

The CityFly Demo program demonstrates how you can use the Spaceball device to interact with geometry using eyepoint control, similar to walk-through movement. When you start CityFly you see a wire-frame model of a city. Moving the PowerSensor ball on the Spaceball device makes the scene move as if the ball were your head. Pushing the PowerSensor ball to the left causes the model to move to the right, pushing the PowerSensor ball to the right causes the model to move to the left, and so on. Rotations are all centered about your viewpoint.

Motion Filter/Constraints

By default, only Y rotations (looking left and right) are allowed in CityFly. To turn on all rotations (left/right, up/down, forward/backward) press Button 4 on a Spaceball 2003 device.

To constrain movement to the single largest screen-based X, Y, or Z axis, activate the Single Axis Filter (Button 3 on the Spaceball 2003 device or select the Single Axis Filter in the SpaceWare Settings window).

Spaceball 2003 Buttons

1.

Toggles translations On/Off. The default is On.

2.

Toggles rotations On/Off. The default is On.

3.

Toggles the Single Axis Filter On/Off. When this is On, only the most dominant force applied to the Spaceball is applied to the object. When this is Off, all motions applied to the Spaceball are applied to the object. The default for this filter is Off.

4.

Toggles roll On/Off. When this is Off, the program filters out any rotation except for left/right rotation.

5.

Lowers the sensitivity of both translations and rotations.

6.

Increases the sensitivity of both translations and rotations.

7.

Not used.

8.

Spaceball 2003A and 2003B: Rezeroes the Spaceball. Instructs the Spaceball to assume a rest position in case of image drift (movement of the select object(s) independent of any pressure on the Spaceball device).

Spaceball 2003 FLX: Resets the view to the default.

Pick

Spaceball 2003A and 2003B: Resets the view to the default. This button is not available on the 2003 FLX.

Spaceball 3003 Buttons

Right

Resets the view to the default.

Left

Toggles the driver menu On/Off.

Keys

Esc: Quits the program.

Widget World

The Widget World demo program demonstrates how you can use the Spaceball device to interact with geometry using object control, similar to typical CAD system movement.

You need to run Widget with a data file for an argument. There is one in %SDK%/demos/widget/dat/widget.dat. Once you select a data file, a wire-frame of the model appears with a small cross roughly in the middle of it. Moving the PowerSensor ball causes the model to move the same way. Pushing the PowerSensor ball to the left moves the model to the left, pushing the PowerSensor ball to the right moves the model to the right, and so on. This is object mode.

The data file format for the Widget World scene is a set of points in X, Y, and Z format. These points are connected by edges to form line segments. The line segments are connected to form polygons. The polygons are rendered to the screen to display a scene. The first two lines on the file are points that define the extents of the scene. Next appear the number points, edges, and polygons. Then, the points, the edges (defined by the numbers of the points they connect), and finally the polygons appear. The polygons contain data fields for the number of edges they contain, a color for rendering the polygon (not currently used), and the list of edges.

sbtest32

This is a very simple program which reads the Spaceball data and prints it to a Win32 Window. It also prints out the button presses and releases as well as zero events.

 

sbtstmfc

This is a very simple program which reads the Spaceball data and prints it to a MFC Window. It also prints out the button presses and releases as well as zero events.
