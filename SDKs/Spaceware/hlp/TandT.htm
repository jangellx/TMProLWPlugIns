Tips & Techniques

Common Problems

  • You receive an error message stating: “ERROR: an OS type must be defined -- e.g. OS_WIN32”. This means that OS_WIN32 has not been defined for your project. To add this definition, go to Build > Settings > C/C++ > Preprocessor. In the Preprocessor Definitions section, add the definition. Adding this to your project settings should clear up the problem.

  • You receive errors with si.h. Double check that OS_WIN32 is defined for your project.

  • You get any one of the following unresolved symbols error messages when linking:

            __imp__timeBeginPeriod
            __imp__timeEndPeriod
            __imp__timeGetTime

    This means you are not linking with winmm.lib. Add winmm.lib to the link libraries in your project settings.

  • SiOpen returns a NULL SiHdl. Check to make sure that a Spaceball is connected and that the SpaceWare driver has been installed. Run one of the demos which ships with the driver to make sure that the driver is configured properly. Next, verify that siappdll.dll and siappdld.dll are in the Windows system directory (or system32 directory for Windows NT). Check that a valid window handle is being passed to SiOpenWinInit and from there into SiOpen. If you are still having problems, try printing out the error message returned from SiOpen as follows:

       MessageBox (hWnd, SpwErrorString 
       (SpwErrorVal),"Spaceball     Error",MB_OK);

    where hWnd is the window you are passing into SiOpenWinInit.

 

Tuning

The Spaceball is designed to be a very intuitive device. To ensure that the Spaceball is as simple to use as possible, it is vital that the mappings and tuning be optimized for your specific application. To accomplish this, you will need to adjust the values of the Spaceball data to match your application.

This is done by multiplying or dividing the input data by a constant value, much the same as you would normally do with mouse or keyboard data. Sometimes it is desirable to provide a sensitivity value which can be adjusted by the user. See the CityFly demo program for an example of this functionality.

Here are a few questions to ask yourself about the tuning:

  1. Can you move along each individual axis without drifting or getting unwanted motion (for example, moving forward and backward without rotating)?
  2. Can you move predictably (rotate 90 or 180 degrees, for example)?
  3. Can you keep the object on screen while moving around?
  4. Can you perform a multi-axis movement (move and turn at the same time)?
  5. Can you mix movements easily (move diagonally, for example)?

Refer to Scoring for more information about fine-tuning the Spaceball device in your application.
