Math Library

This portion of the Spaceball library contains some simple mathematical functions that Spacetec finds useful when integrating the Spaceball into an application.

 
Math Library Philosophy

Every graphics engine eventually requires a world-to-eye transformation matrix to convert an application's data from its coordinate space to eye space immediately prior to projection to the 2D screen.

Eye space is what you see when you look at an image on a computer monitor. It is at this point that you interact with the Spaceball. You push the ball to the left, up, down, etc. Therefore your interaction takes place in eye space. You are telling the graphics program to take the image on the screen and move it left, up, down, etc.

Eye space and the world-to-eye matrix form a consistent interface for applying the Spaceball input into any program.

In the spirit of friendship and happy side-effectless programming, all functions that have original and result arguments are written to "not care" if the original and the result are the same array.

 
The Coordinate System

The world_to_eye matrix must be the true world_to_eye (scale is ok). If it is composed of several matrices (such as [Trans][Rot][Scale]), the composition must be passed in and then decomposed by the caller after Library modification. It is better to change the code to use one world_to_eye matrix.

It is the caller's responsibility to apply Spaceball data in the correct coordinate system. If you want to move something in world (or other) space, transform the Spaceball data from the eye coordinate system to the target coordinate system.

 

Math Library Structures/Definitions

Matrix

typedef float Matrix[4][4];

Description

The Matrix data type is used throughout the SpaceWare Math Library. It is a 4x4 float array in which the rotation is held in the upper right 3x3 matrix; the translations are held in the 4th row, columns 1-3. The matrix may be scaled. The last column is:

[0
0
0
1]

The matrix is stored in row major order.

The matrix therefore is:

[ [ Side Vector ] 0 ]
[ [ Up Vector ] 0 ]
[ [ Toward Vector ] 0 ]
[ [ Translation ] 1 ]

This is the form common to computer graphics literature. It is the transpose of the form common in the robotics literature.

This type will not be defined if the GL header file is included before spwslim.h.

SPW_InvertMatrix

void SPW_InvertMatrix(result,m)

Matrix result; 

(w/o) 

Matrix m; 

(r/o) 

Mathematics
Transposes the rotation part of the matrix. It inverts the last row by negating the translation and dotting each element with its respective (untransposed) row.

This method does not work with any matrix - it must be a standard transformation matrix. This is not doing gaussian elimination, but taking advantage of some special properties of transformation matrices.

Description
Inverts an orthogonal transformation matrix.

SPW_MatrixDeterminant

float SPW_MatrixDeterminant(m)

Matrix m; 

(w/o) 

matrix

 

Returns
Determinant of the upper 3x3 part of m

Mathematics
The determinant of A is a combination of row i and the cofactors of row i:

det A = ai1Ai1 + ai2Ai2 + ... + ainAin

The cofactor Aij is the determinant of Mij with the correct sign:

Aij = (-1)**i+j det Mij

Mij is formed by deleting row i and column j of A.

The determinant of an orthornormal matrix is +1.0.

The determinant gives the volume of the box enclosed by the vectors described by the matrix.

Description
Calculates the determinant of the upper 3x3 part of a matrix (rotations only).

SPW_MatrixToLookAtUpVector

void SPW_MatrixToLookAtUpVector(looking_from_pt,looking_at_pt,up_vector,m)

float looking_from_pt[3]; 

(w/o) 

point in world coordinates where you are 

float looking_at_pt[3]; 

(w/o) 

point in world coordinates at which you are looking (sometimes called the reference point) 

float up_vector[3]; 

(w/o) 

"up" direction in world coordinates. This is a vector pointing in the direction that is directly up (the positive y axis) in the eye coordinate system 

Matrix m; 

(r/o) 

view matrix (world_to_eye) from which to extract parameters 

Description
Returns "look at" parameters from an existing world_to_eye matrix. These looking_at_pt may be different from the looking_at_pt passed to SPW_LookAtUpVectorToMatrix because only the direction from the looking_at_pt to the looking_from_pt is used to calculate the matrix, the actual distance between these points is not in the matrix (so it can not be retrieved). As a result the looking_at_pt that is returned from this function is one unit length away from the looking_from_pt. It is in the right direction, just probably not far enough along. This should not make a difference.
